Git常用命令

9月 18 2019

  1. 用户配置

    • git config –global user.name w4n9hu1
    • git config –global user.email wanghui89757@gmail.com

  2. 版本控制

    • git init
    • git add Git.md
    • git commit -m “add Git.md”
    • git status
    • git diff
    • git log 查看提交历史
    • git reset –hard “commit id” 恢复到指定版本
    • git reflog 查看历史命令
    • git checkout – Git.md 放弃工作区的修改
    • git reset HEAD Git.md 撤销缓存区修改
    • git rm t.md 从版本库中删除文件
    • git checkout – t.md 从版本库中恢复文件
    • git tag name 默认标签是打在最新提交的commit上
    • git tag 查看所有标签
    • 分支
      • git branch dev 创建分支dev
      • git branch 显示所有分支
      • git checkout dev 切换到分支dev
      • git checkout -b develop origin/develop 切换到远程分支develop
      • git merge dev 把dev分支合并到master
      • git branch -d dev 删除dev分支
      • –no-ff 禁用Fast forward模式，merge时生成一个新的commit，从分支历史上就可以看出分支信息。
      • git push origin dev 推送其他分支，Github自动创建该分支

  3. 远程仓库Github

    • ssh-keygen -t rsa -C wanghui89757@gmail.com 创建SSH Key
    • ~/.ssh/id_rsa.pub
    • Github>settings>New SSH Key
    • git remote add origin https://github.com/W4n9Hu1/Articles.git 关联远程仓库，origin默认远程仓库名
    • git push -u origin master 本地库推送到远程库
    • git push origin master 把本地master分支推送到Github
    • git clone https://github.com/W4n9Hu1/Articles.git 从远程仓库克隆到本地
    • git branch dev 创建分支dev
    • git checkout dev 切换到分支dev
    • git remote -v 查看远程库信息
    • git pull 从远程拉取分支
    • git branch -r 查看远程分支
    • git push origin –delete dev 删除远程分支

  4. 其他

    • git add -A . 添加所有改变的文件
    • git add . 添加新文件和编辑过的文件不包括删除的文件
    • git add -A 添加所有内容
    • git add -u 添加编辑或者删除的文件，不包括新添加的文件。
    • -u 把本地的master分支内容推送的远程新的master分支，并且把两者关联起来
    • git log –graph –pretty=oneline –abbrev-commit 查看分支合并情况
    • 提交所有文件
    • mkdir 新建一个子目录
    • rm 删除一个目录中的一个或多个文件或目录，也可以将某个目录及其下属的所有文件及其子目录均删除掉
    • 针对特定项目使用不同的用户名称与邮件地址时，可以在那个项目目录下运行没有 –global 选项的用户配置命令来配置
    • Repository 版本库
